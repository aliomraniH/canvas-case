1 · Highlights

  • The trial paper is now applied, not just cited. Clinicians never have to convert kilograms by hand: per-patient figures are shown in the patient's own unit, and the trial's population means are shown in both units, clearly labeled as population.
  • Dual-metric headline. Current loss now reads as BOTH %TBWL and absolute change in the patient's display unit — e.g. −12.1% TBWL (−26.6 lb from 220 lb baseline) — both computed from the patient's own baseline and latest weight, never from a trial figure.
  • Milestone lines carry the weight. The 5/10/15% reference lines now label the patient-unit target alongside the percent: 5% — 209 lb.
  • Trial disclosure in both units. Where a trial reports an absolute mean (SCALE), the ⓘ panel shows 8.4 kg ≈ 18.5 lb at the 106.2 kg trial baseline, explicitly a population figure — so the lb equivalent is visible without conversion AND cannot be mistaken for a patient target. STEP-1/SURMOUNT-1 carry percent only, so no kg was invented.
  • Basis wording corrected. The SCALE band basis is the full analysis set with 56-week LOCF imputation — not “trial completers” (LOCF carries dropouts forward, so it includes them). A copy-only fix; band geometry unchanged.

3 · Bugs found & fixed

v0.2.5 (Task A)The SCALE disclosure described the band as “LOCF, trial completers”, which is self-contradictory — LOCF imputes dropouts by carrying their last value forward, so the population is the full analysis set, not completers. Corrected the disclosure, rationale, reference header, and the v0.2.4 report. Repo-wide grep -i completer is clean of stale claims.
v0.2.5 (structural)The kg→lb factor was a bare literal (2.20462) — the same loose-literal pattern behind the earlier kg-vs-% class of errors. One exact constant (KG_PER_LB = 0.45359237), every kg factor derived from it, round-trip tested to 1e-9, and a test that fails if any stray conversion literal reappears in the source.
No plugin behavior regressions: band geometry and all prior pinned values are byte-identical; the kg constant gained precision (sub-milligram effect on kg-sourced weights, within every existing tolerance).

4 · Observations & architectural findings

  • Percent is portable; absolute weight is not. The design principle now stated verbatim in the rationale: per-patient targets are always the published percent applied to the patient's own baseline in their own unit; trial kg figures are population means and are never converted into an individual target. This is what keeps the dual-unit display honest.
  • One conversion constant is a structural control, not a style choice. Centralizing the factor and adding a “no stray literal” test removes the substrate the kg-vs-% bug class grew from.
  • A “no stray literal” test must strip comments first — prose that cites the constant's value (“0.45359237 kg by definition”) is documentation, not a stray literal. Caught on first run.

5 · Future version considerations

  • Predominant-unit display: display unit is currently lb (the normalization target, always coherent). Selecting a patient's predominant recorded unit (e.g. show kg for a kg-charted patient) is a v-next refinement.
  • Absolute means for STEP-1/SURMOUNT-1: if their published kg means + baselines are sourced and added to the reference, the dual-unit population line extends to them automatically.
  • CDF responder-rate markers (SCALE anchors shipped as data in v0.2.4) and the SURMOUNT-1 estimand reconciliation remain open.
  • Milestone crossed semantics on regain; mid-series medication changes; the v0.2-deferred dose-titration markers / week-12 banner / BMI transitions / extrapolation.
